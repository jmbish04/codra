# Required observability endpoints

Every Cloudflare Worker project under cloudflare-jedi MUST consistently expose these endpoints. They are served by the Hono app inside Astro middleware (see `references/hono-openapi.md` for the architecture).

## The mandatory set

| Path | Purpose | Format |
|---|---|---|
| `/openapi.json` | Live OpenAPI 3.1.0 spec for every Hono route | application/json |
| `/swagger` | Swagger UI rendering of `/openapi.json` | text/html |
| `/scalar` | Scalar API reference (richer than Swagger) | text/html |
| `/health` | On-demand live system health scan | application/json |
| `/context` | LLM-ingestible Markdown context dump | text/markdown |
| `/docs` | Comprehensive docs site (Astro-rendered) | text/html (Astro page) |

All six are **always present**, **always dynamic** (never hardcoded), and **always linked from the Navbar**.

## `/openapi.json`

Generated by `@hono/zod-openapi` from the route definitions. Must be OpenAPI 3.1.0 (not 3.0.x — 3.1 supports JSON Schema 2020-12 which Drizzle-zod emits).

```typescript
// src/hono/index.ts
app.doc('/openapi.json', {
  openapi: '3.1.0',
  info: { title: 'My App API', version: '1.0.0' },
  servers: [{ url: '/' }],
})
```

Never hand-edit. Never hard-code. The spec must come from the live route definitions.

## `/swagger` and `/scalar`

```typescript
import { apiReference } from '@scalar/hono-api-reference'
import { swaggerUI } from '@hono/swagger-ui'

app.get('/scalar', apiReference({ spec: { url: '/openapi.json' } }))
app.get('/swagger', swaggerUI({ url: '/openapi.json' }))
```

Both reference the dynamic `/openapi.json` — keep them in sync automatically.

## `/health`

Returns the live system status. The shape:

```typescript
type HealthResponse = {
  status: 'ok' | 'degraded' | 'down'
  timestamp: string
  checks: {
    d1: 'ok' | 'fail' | 'unknown'
    ai_gateway: 'ok' | 'fail' | 'unknown'
    workers_ai: 'ok' | 'fail' | 'unknown'
    r2?: 'ok' | 'fail' | 'unknown'
    kv?: 'ok' | 'fail' | 'unknown'
    durable_objects?: Record<string, 'ok' | 'fail' | 'unknown'>
    agents?: Record<string, 'ok' | 'fail' | 'unknown'>
  }
  version: string  // from package.json or git commit short hash
}
```

Each check runs a cheap probe at request time:
- D1: `SELECT 1` against the bound database.
- AI Gateway: HEAD or quick prompt to a free model.
- Workers AI: lightweight model probe.
- R2: `headObject` on a known sentinel key.
- KV: `get` on a known sentinel key.
- Durable Objects: `fetch` against each DO's `/health` endpoint.
- Agents: each agent's `health.ts` method.

Sample handler:

```typescript
// src/hono/routes/health.ts
import { createRoute, OpenAPIHono } from '@hono/zod-openapi'
import { z } from 'zod'

const healthRouter = new OpenAPIHono<{ Bindings: Env }>()

const route = createRoute({
  method: 'get',
  path: '/health',
  responses: { 200: { description: 'health', content: { 'application/json': { schema: HealthResponseSchema } } } },
})

healthRouter.openapi(route, async (c) => {
  const checks = {
    d1: await probeD1(c.env.DB).then(() => 'ok' as const).catch(() => 'fail' as const),
    ai_gateway: await probeAIGateway(c.env.AI).then(() => 'ok' as const).catch(() => 'fail' as const),
    // ... rest of probes (write them all, no // ... rest patterns)
  }
  const allOk = Object.values(checks).every((v) => v === 'ok')
  return c.json({
    status: allOk ? 'ok' : 'degraded',
    timestamp: new Date().toISOString(),
    checks,
    version: c.env.GIT_SHA ?? 'dev',
  })
})

export { healthRouter }
```

Wire into the main Hono app:

```typescript
app.route('/health', healthRouter)
```

## `/context`

Markdown dump of the project's operational state, optimized for LLM ingestion (a future agent loading this page should be able to bootstrap full understanding from it).

Sections:

```markdown
# Project Context

## Stack
- Cloudflare Workers + Astro SSR + Hono + D1 + Drizzle + ...

## Bindings (from wrangler.toml)
- D1: ${BINDING_NAMES}
- KV: ${BINDING_NAMES}
- ...

## Schemas
- ${TABLE_LIST_WITH_BRIEF_DESCRIPTIONS}

## Hono routes
- ${ROUTE_LIST}

## Agents
- ${AGENT_LIST_WITH_METHODS}

## Providers
- ${PROVIDER_LIST}

## Health
- See `/health` for live status.

## Docs
- See `/docs` for full pages.
- See `/openapi.json` for the API spec.
```

Generate this dynamically — read from the live binding inventory + route registry. Don't hand-author.

Sample handler:

```typescript
// src/hono/routes/context.ts
healthRouter.get('/context', async (c) => {
  const markdown = await buildContextMarkdown(c.env)
  return c.text(markdown, 200, { 'Content-Type': 'text/markdown' })
})
```

## `/docs`

An Astro-rendered docs site with TSX page modules. Structure:

```
src/pages/docs/
├── index.astro              # Index linking to all pages
├── [...slug].astro          # Dynamic content
└── _data/                   # Markdown content files
```

Includes:
- API reference (linked to `/scalar`).
- Architecture diagrams.
- Onboarding guide.
- Runbooks for common ops (deploy, rollback, debug).

## Navbar linkage

Every page in the app includes `<Navbar />` (see `references/astro-frontend.md`). The Navbar must link to all six observability endpoints:

```astro
---
// src/pages/_components/Navbar.astro
---
<nav class="...">
  <a href="/">Home</a>
  <a href="/docs">Docs</a>
  <div class="ml-auto flex gap-4 text-sm text-muted-foreground">
    <a href="/openapi.json">OpenAPI</a>
    <a href="/scalar">Scalar</a>
    <a href="/swagger">Swagger</a>
    <a href="/health">Health</a>
    <a href="/context">Context</a>
  </div>
</nav>
```

(Style this per the Monolith spec — no traditional borders, use `ring-1 ring-border/40` on the navbar bottom if separation is needed.)

## Audit check

Before any deploy, run a smoke test against all six endpoints:

```bash
for path in /openapi.json /swagger /scalar /health /context /docs; do
  echo "=== $path ==="
  curl -sSf "http://localhost:8787${path}" | head -c 200
  echo
done
```

Each should return 200 with non-empty content. Add this to the project's `package.json` as `pnpm run smoke`.

## Anti-patterns

- ❌ Hardcoding `/openapi.json` instead of generating it from route defs.
- ❌ `/health` returning `{ status: 'ok' }` without actually probing bindings.
- ❌ `/context` written by hand (it MUST be live).
- ❌ Skipping the Navbar links because "the URL is in the address bar."
- ❌ Hiding `/health` behind auth in production — it's the operations entrypoint, keep it public (or behind a cheap shared secret if you must).
